## Các phần bắt buộc theo yêu cầu 3.2

### 1. Wireframe
Nhóm tạo wireframe cho các màn hình chính: Login, Trang chủ/Dự đoán, Lịch sử dự đoán, Quản lý thuốc/Nhóm thuốc, Hồ sơ cá nhân.

### 2. User Flow
Luồng chính:
Người dùng mở website → Đăng nhập → Vào trang chủ → Nhập triệu chứng → Xem kết quả dự đoán → Lưu lịch sử → Xem lại lịch sử.

Luồng quản trị:
Admin đăng nhập → Quản lý thuốc/Nhóm thuốc → Thêm/Sửa/Xóa nhóm thuốc → Dữ liệu được lưu vào database.

### 3. Prototype
Các màn hình Figma được liên kết bằng Prototype: Login → Dashboard → Quản lý thuốc → Hồ sơ → Lịch sử.

### 4. Mockup
Mockup sử dụng giao diện MediPredict AI, màu xanh y khoa, card bo góc, sidebar điều hướng và form nhập liệu rõ ràng.

### 5. Design System
- Font: Inter
- Màu chính: xanh y khoa
- Button: bo góc, nền xanh, chữ trắng
- Input/Textarea: bo góc, border nhẹ
- Card: nền trắng hoặc nền tối theo theme, có shadow nhẹ

## Ghi chú triển khai
Ban đầu nhóm có giao diện prototype để kiểm thử API. Sau đó nhóm thiết kế UI/UX trên Figma và cập nhật frontend theo giao diện Figma. Backend Flask/API giữ nguyên để không phá vỡ tiến độ Jira/GitHub.
